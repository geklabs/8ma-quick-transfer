Transferencia reanudable

¿Cómo reanudar una transferencia de archivos grandes interrumpida?

Una interrupción de red, actualización de página o cierre temporal del navegador no siempre significa empezar de nuevo. En navegadores compatibles, 8ma Quick Transfer conserva la parte completada en el dispositivo receptor y reanuda desde un punto de control verificado.

La respuesta corta

En el dispositivo emisor original, continúa el envío anterior y elige el mismo archivo. En el dispositivo receptor, vuelve a abrir el enlace original y selecciona “Reanudar descarga”.

Reanudar en tres pasos

  1. Vuelve a 8ma Quick Transfer en el emisor y elige el archivo original del envío anterior.
  2. Vuelve a abrir el enlace original o la página del código de recogida en el dispositivo receptor.
  3. Selecciona “Reanudar descarga”. La transferencia continúa después de verificar la parte guardada.

¿Qué bytes no necesitan enviarse de nuevo?

La parte verificada ya guardada en el dispositivo receptor no se envía de nuevo. Si no coincide con el archivo original, el punto de control no válido se descarta para que el receptor no conserve un archivo dañado.

¿Por qué el emisor elige el archivo de nuevo?

Después de que una página se cierra o se actualiza, el navegador no puede leer un archivo local sin permiso. El emisor confirma el archivo original de nuevo, y su nombre, tamaño y fecha de modificación deben coincidir con el envío anterior.

¿Cómo evitar otra interrupción?

  • Mantén ambas páginas abiertas y evita que los dispositivos entren en suspensión.
  • Usa la versión actual de Chrome o Edge en escritorio para archivos muy grandes.
  • Prefiere una conexión estable; en la misma WiFi, el envío cercano también está disponible.
