Resumable transfer

How do you resume an interrupted large file transfer?

A network interruption, page refresh, or temporarily closed browser does not always mean starting over. In supported browsers, 8ma Quick Transfer keeps the completed portion on the receiving device and resumes from a verified checkpoint.

The short answer

On the original sending device, continue the previous share and choose the same file. On the receiving device, reopen the original link and select “Resume download.”

Resume in three steps

  1. Return to 8ma Quick Transfer on the sender and choose the original file for the previous share.
  2. Reopen the original link or pickup-code page on the receiving device.
  3. Select “Resume download.” Transfer continues after the saved portion has been checked.

Which bytes do not need to be sent again?

The verified portion already saved on the receiving device is not sent again. If it does not match the original file, the invalid checkpoint is discarded so the receiver does not keep a damaged file.

Why does the sender choose the file again?

After a page closes or refreshes, the browser cannot read a local file without permission. The sender confirms the original file again, and its name, size, and modification time must match the previous share.

How can you avoid another interruption?

  • Keep both pages open and prevent the devices from sleeping.
  • Use the current desktop Chrome or Edge for very large files.
  • Prefer a stable connection; on the same WiFi, Nearby Sharing is also available.
