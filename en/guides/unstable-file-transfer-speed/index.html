Transfer speed

Why is file transfer speed unstable, and how can you improve it?

A transfer can only run as fast as the slowest current constraint across the sending device, receiving device, and their available network. When any of those conditions change, the displayed speed can change too.

The short answer

Use the same WiFi when possible, keep both devices close to a strong access point, pause competing downloads, prevent sleep, keep both pages open, and use a current Chrome or Edge browser. For maximum speed, send to one receiver at a time.

First, check Mbps versus MB/s

Network rates are normally shown in megabits per second (Mbps), while file transfer speed is often shown in megabytes per second (MB/s). Eight bits make one byte, so a 100 Mbps connection equals 12.5 MB/s before normal overhead. It does not mean 100 MB/s.

Eight ways to make a large file transfer faster and steadier

  1. Use the same local network when possible. Open 8ma Quick Transfer on both devices and use Nearby Sharing instead of sending across different networks.
  2. Improve the WiFi signal. Move both devices closer to the access point and avoid weak-signal rooms or crowded public WiFi.
  3. Pause competing traffic. Video streaming, backups, game downloads, and other file transfers can reduce the bandwidth available now.
  4. Keep both devices awake. Plug in laptops and phones for very large files, and prevent power-saving mode from suspending the browser.
  5. Keep both browser pages open. Do not close, refresh, or navigate back during the transfer.
  6. Use a current browser. Desktop Chrome or Edge is recommended for very large files and supports files up to 100 GB.
  7. Check both devices. A busy device or slow local read/write operation can limit the whole transfer even when the network is fast.
  8. Use one receiver for maximum speed. A share supports up to three receivers, but simultaneous receivers share the sender's available resources.

Why can speed start fast and then drop?

A short burst can use bandwidth that is only temporarily available. Speed may fall when other traffic starts, WiFi signal changes, a device enters power-saving mode, or local read/write work becomes the slowest part. The useful number is the sustained average over the whole transfer, not the highest momentary value.

What if the transfer is interrupted?

In supported browsers, the completed portion can remain on the receiving device. Reopen the share, confirm the same original file on the sending device, and continue from the verified checkpoint instead of starting the entire file again.
