전송 속도

파일 전송 속도가 불안정한 이유와 어떻게 개선할 수 있을까요?

전송은 보내는 기기, 받는 기기, 그리고 사용 가능한 네트워크 전반에서 현재 가장 느린 제약 조건만큼만 빠르게 실행될 수 있습니다. 이러한 조건 중 하나라도 변경되면 표시되는 속도도 변경될 수 있습니다.

짧은 답변

가능하면 동일한 WiFi를 사용하고, 두 기기를 강력한 액세스 포인트 근처에 두고, 경쟁 다운로드를 일시 중지하고, 절전 모드를 방지하고, 두 페이지를 모두 열어 두고, 최신 Chrome 또는 Edge 브라우저를 사용하세요. 최대 속도를 원한다면 한 번에 하나의 수신기로 보내세요.

먼저, Mbps와 MB/s의 차이를 확인하세요

네트워크 속도는 일반적으로 초당 메가비트(Mbps)로 표시되는 반면, 파일 전송 속도는 종종 초당 메가바이트(MB/s)로 표시됩니다. 8비트가 1바이트를 구성하므로, 100 Mbps 연결은 일반적인 오버헤드를 제외하면 12.5 MB/s와 같습니다. 100 MB/s를 의미하는 것이 아닙니다.

대용량 파일 전송을 더 빠르고 안정적으로 만드는 8가지 방법

  1. 가능하면 동일한 로컬 네트워크를 사용하세요. 두 기기에서 8ma Quick Transfer를 열고 다른 네트워크를 통해 보내는 대신 근거리 공유를 사용하세요.
  2. WiFi 신호를 개선하세요. 두 기기를 액세스 포인트에 더 가까이 옮기고 신호가 약한 방이나 혼잡한 공용 WiFi를 피하세요.
  3. 경쟁 트래픽을 일시 중지하세요. 비디오 스트리밍, 백업, 게임 다운로드 및 기타 파일 전송은 현재 사용 가능한 대역폭을 줄일 수 있습니다.
  4. 두 기기를 모두 깨어 있게 유지하세요. 매우 큰 파일의 경우 노트북과 휴대폰을 전원에 연결하고, 절전 모드로 인해 브라우저가 일시 중단되지 않도록 하세요.
  5. 두 브라우저 페이지를 모두 열어 두세요. 전송 중에 페이지를 닫거나, 새로 고침하거나, 뒤로 탐색하지 마세요.
  6. 최신 브라우저를 사용하세요. 매우 큰 파일에는 데스크톱 Chrome 또는 Edge를 권장하며 최대 100GB 파일을 지원합니다.
  7. 두 기기를 모두 확인하세요. 사용 중인 기기나 느린 로컬 읽기/쓰기 작업은 네트워크가 빠르더라도 전체 전송을 제한할 수 있습니다.
  8. 최대 속도를 위해 하나의 수신기를 사용하세요. 공유는 최대 3개의 수신기를 지원하지만, 동시 수신기는 발신자의 사용 가능한 리소스를 공유합니다.

속도가 빠르게 시작되었다가 떨어지는 이유는 무엇인가요?

짧은 순간에는 일시적으로만 사용 가능한 대역폭을 사용할 수 있습니다. 다른 트래픽이 시작되거나, WiFi 신호가 변경되거나, 기기가 절전 모드로 전환되거나, 로컬 읽기/쓰기 작업이 가장 느린 부분이 되면 속도가 떨어질 수 있습니다. 유용한 수치는 가장 높은 순간 값이 아니라 전체 전송에 걸친 지속 평균입니다.

전송이 중단되면 어떻게 하나요?

지원되는 브라우저에서는 완료된 부분이 수신 기기에 남아 있을 수 있습니다. 공유를 다시 열고, 보내는 기기에서 동일한 원본 파일을 확인한 다음, 전체 파일을 다시 시작하는 대신 확인된 체크포인트에서 계속 진행하세요.
