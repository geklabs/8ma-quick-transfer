재개 가능한 전송

중단된 대용량 파일 전송을 어떻게 재개하나요?

네트워크 중단, 페이지 새로고침 또는 일시적인 브라우저 종료가 항상 처음부터 다시 시작해야 한다는 의미는 아닙니다. 지원되는 브라우저에서 8ma Quick Transfer는 수신 기기에 완료된 부분을 유지하고 검증된 체크포인트에서 재개합니다.

간단한 해결책

원래 보내던 기기에서 이전 공유를 계속 진행하고 동일한 파일을 선택하세요. 수신 기기에서는 원래 링크를 다시 열고 "다운로드 재개"를 선택하세요.

세 단계로 재개하기

  1. 발신자 측에서 8ma Quick Transfer로 돌아가 이전 공유를 위한 원본 파일을 선택하세요.
  2. 수신 기기에서 원래 링크 또는 픽업 코드 페이지를 다시 여세요.
  3. "다운로드 재개"를 선택하세요. 저장된 부분을 확인한 후 전송이 계속됩니다.

다시 보낼 필요가 없는 데이터는 무엇인가요?

수신 기기에 이미 저장되어 검증된 부분은 다시 보내지 않습니다. 원본 파일과 일치하지 않으면 유효하지 않은 체크포인트는 폐기되므로 수신자가 손상된 파일을 보관하지 않습니다.

발신자가 파일을 다시 선택해야 하는 이유는 무엇인가요?

페이지가 닫히거나 새로고침되면 브라우저는 허가 없이 로컬 파일을 읽을 수 없습니다. 발신자는 원본 파일을 다시 확인하며, 파일 이름, 크기 및 수정 시간이 이전 공유와 일치해야 합니다.

또 다른 중단을 방지하려면 어떻게 해야 하나요?

  • 양쪽 페이지를 열어 두고 기기가 절전 모드로 전환되지 않도록 하세요.
  • 매우 큰 파일의 경우 최신 데스크톱 Chrome 또는 Edge를 사용하세요.
  • 안정적인 연결을 선호하며, 동일한 WiFi에서는 주변 공유도 사용할 수 있습니다.
